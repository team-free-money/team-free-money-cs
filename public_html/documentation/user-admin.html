User Story

As a techie and PC Gamer, I want to browse optimized websites with interactivity features.

Use Case

Vanna is at work, doing an inspection on the website. On these days, she goes through each page, looking for typos and ensuring all links are working correctly. She notices there is a broken link for one of the scholarships. Vanna knows that a broken link can negatively affect the user's experience, so she tries to find out why the link broke. Upon investigation, she realizes she had entered the wrong URL.

Interaction Flow

  1. Vanna opens the Home page of the website.
  2. The website opens a log in or browse option.
  3. Vanna logs in using her admin credentials.
  4. The website loads the Home page.
  5. Vanna begins scrolling the Home page, looking for typos and ensuring all links are working properly. She then clicks to the next page.
  6. The website loads the second page.
  7. Vanna looks for typos and clicks a link.
  8. The website opens up a 404 error page.
  9. Vanna presses the go back arrow on the top of the browser.
  10. The website loads the previous page.
  11. Vanna investigates to fix the broken link.
